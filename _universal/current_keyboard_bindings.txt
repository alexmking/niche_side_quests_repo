MIDDLE-MAN & PLAYGROUND Fn-mini-groups of shortcuts
====================================================================================================================================================
====================================================================================================================================================
MIDDLE-MAN shortcuts expl: these shortcuts are never triggered myself, but rather for the hacky workarounds where 1 app/tool has to pass execution onto another by sending keystrokes that trigger it. 
    EX: my fancy-chrometab shortcut: hyper-d triggers the BTT shortcut which switches to chrome, then BTT sends keystrokes (the middle-man shortcut) which trigger the chrome-extensions 'jump-to-tab-by-URL' and so on. 
    TIP: bc I never actually have to trigger these shortcuts myself, they can be super-awkward and impractical-to-press shortcuts since it's simply the keystrokes() which we hardcode beforehand
    TIP: also you want to avoid using any actual useful/valuable shortcuts for these since they'll be wasted real-estate that you'll never manually trigger anyway. 
PLAYGROUND shortcuts expl: these are groups of keys that aren't always open/avail for quick-adding a new-shortcut/command that you want to test/try-out before officially giving it an actual binding in one of the MAIN FOUR categories
    EX: how I was using the ALT-SHIFT-NUMPAD 3x3 group, I'd add a command there first, try it out, and then after a while try moving the shortcut to the official main 4 categories

=============tmp-CLARIFICATION-OF-TERMS=============
key-binding:    defines a mapping (of binding) of a hotkey to an action/command. So "(ALT + T)  ==  Open Terminal" is a key-binding, which maps ALT+T to OpenTerminal command
shortcut:       interchangeable with key-binding, it's just a more user-friendly term than key-binding 
hotkey:         a combination of keys/mods that triggers a command. So for our shortcut "(ALT + T)  ==  Open Terminal", the hotkey would be (ALT +T)
